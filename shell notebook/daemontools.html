daemontools 文档
进程监控工具 daemontools

1 常用命令

$ sudo svc -d /etc/service/applog2es   # 关闭进程
$ sudo svstat /etc/service/applog2es   # 查看 service 运行状态
/etc/service/applog2es: up (pid 14333) 2986 seconds
$ sudo svstat /etc/service/sfServer_online_run
/etc/service/sfServer_online_run: down 29651 seconds, normally up
$ sudo svc -u /etc/service/applog2es   # 启动进程
$ sudo svc -iu /etc/service/applog2es  # 向进程发 SIGINT，再启动进程
$ sudo svc -ku /etc/service/applog2es  # 向进程发 SIGKILL，再启动进程
$ sudo svc -dx /etc/service/applog2es  # 移除指定 service，不再守护了（实测无效，测试结果是子进程启动了）。之后可以使用 sudo rm /etc/service/applog2es 删除软链接

svc 选项有

  • -u Up. If the service is not running, start it. If the service stops, restart it.
  • -d Down. If the service is running, send it a TERM signal and then a CONT signal. After it stops, do not restart it.
  • -o Once. If the service is not running, start it. Do not restart it if it stops.
  • -p Pause. Send the service a STOP signal.
  • -c Continue. Send the service a CONT signal.
  • -h Hangup. Send the service a HUP signal.
  • -a Alarm. Send the service an ALRM signal.
  • -i Interrupt. Send the service an INT signal.
  • -t Terminate. Send the service a TERM signal.
  • -k Kill. Send the service a KILL signal.
  • -x Exit. supervise will exit as soon as the service is down. If you use this option on a stable system, you're doing something wrong; supervise is designed to run forever.

2 知识点

  • Daemontools 是 svscanboot，svscan，supervise，svc，svok，svstat 等一系列工具的合集
  • svscanboot 是 svscan 的开机启动程序
  • svscan
    • svscan 为当前目录的每个子目录启动一个监控进程 supervise，最多可达 1000 个子目录
    • 每个子目录下都会有一个名为 run 的 shell 脚本，用来启动服务
    • 每隔 5s，svscan 再次检查当前目录下的子目录。如果它看到一个新的子目录，它会启动一个新的监控进程
  • supervise
    • supervise 是一种进程监控工具，其功能是当监控的指定进程消亡时，重启进程
    • 监控时，需要为 supervise 指定一个目录(该目录设为 the_dir)，the_dir 用来存放服务启动脚本等
    • supervise 目录：supervise 会在 the_dir 目录下创建名为 supervise 的目录，其中有文件：
      1. control 管道文件。supervise 通过给该管道发信号来控制子进程。例如：
        • echo 'd' > contorl 停掉子进程
        • echo 'u' > contorl 启动子进程
        • echo 'k' > control 向子进程发送 SIGKILL 信号（随后该子进程又会被 supervise 重新拉起来）
      2. lock 文件锁，防止启动多个子进程
      3. ok 通过此文件判断 supervise 进程状态
      4. status supervise 用来存放一些信息，比如子进程的 pid 等。使用命令 od -An -j12 -N2 -tu4 status 可查看子进程 pid

例如：如下进程结构

$ tree -N /etc/service
/etc/service
├── auth_doki_run -> /opt/zeus/service/auth/auth_doki_run/
├── did_doki_run -> /opt/zeus/service/run/did_doki_run/
├── image_uploader_doki_run -> /opt/zeus/service/image_uploader/image_uploader_doki_run
└── utdid_run -> /opt/zeus/run/utdid_run

$ cd did_doki_run && tree -N .
.
├── real_run
├── run           # run 为启动服务的 shell 脚本
└── supervise     # 写好启动脚本后，svscan 会调用 supervise 监控进程，supervise 会自动创建该目录
    ├── control
    ├── lock
    ├── ok
    └── status

$ cat run
#!/bin/sh

who=$(id -un)
ulimit -n 655350
if [ $who = 'root' ]; then
    exec /usr/bin/setuidgid zeus ./real_run
elif [ $who = 'zeus' ];then
    exec ./real_run
else
    echo "not root"
fi

3 注意事项

  • 被管理的进程不能以 daemon 形式运行，例如 nginx.conf 必须关闭 daemon， daemon off
  • 不要在 service 文件夹建任何目录，service 目录只存放一些 symbol link
